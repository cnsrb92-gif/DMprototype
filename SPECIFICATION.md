## 1. 프로젝트 개요

이돌라 세계관의 캐릭터 3명과 실시간 메신저 DM을 주고받는 시뮬레이션 앱.
iPhone 형태의 모바일 UI(390×844px) 안에서 채팅, 인스타 피드, 프로필을 오가며 캐릭터와의 관계를 발전시킨다.

### 핵심 루프

```
채팅 → 관계도 상승 → 텐션 게이지 충전 → 데이트 → 피드 포스팅 → 댓글 상호작용
  ↑                                                                    ↓
  └──────────────── 선톡 / 질투 DM / 읽씹 반응 ◄─────────────────────────┘
```

### AI 연동

- **모델**: Google Gemini 2.0 Flash (`gemini-2.0-flash`)
- **용도**: 채팅 응답 생성, 피드 댓글 생성, 질투/부러움 반응 생성
- **API 키**: 환경변수(`VITE_GEMINI_API_KEY`) 또는 앱 내 입력 → localStorage 저장

## 3. 채팅 시스템

### 3.1 AI 응답 생성

Gemini API에 전달하는 프롬프트 구성:

| 요소 | 내용 |
|------|------|
| 캐릭터 정보 | 이름, 정체성, 성격, 말투 |
| 관계 레벨 | 현재 관계도 기반 설명 |
| 관계 점수 | 0~100 수치 |
| 에너지 레벨 | 시간대별 (Low/Normal/High) |
| 텐션 수치 | 0~100 |
| 대화 이력 | 최근 30개 메시지 |

### 3.2 메시지 분할 (`||SPLIT||`)

AI 응답이 `||SPLIT||` 구분자를 포함하면 여러 말풍선으로 나뉘어 표시된다.

- **에너지 High** (10:00~13:59): 긴 메시지, 2~3개 분할 권장
- **에너지 Normal**: 보통 길이
- **에너지 Low** (22:00~06:59): 짧은 단일 메시지

분할된 메시지는 700ms + 랜덤 0~300ms 간격으로 순차 표시.

### 3.3 점수 파싱

AI 응답에서 다음 마커를 파싱:
- `||SCORE:±n||` → 관계도 변화량
- `||TENSION:±n||` → 텐션 변화량

### 3.4 메시지 데이터 구조

```js
{
  id: Number,           // Date.now() 기반 고유 ID
  sender: 'user' | 'bot' | 'system',
  text: String,
  timestamp: String,    // "방금", "1분 전" 등
  type: 'text' | 'system' | 'image'
}
```

### 3.5 늦은 응답 처리

- 놓침 횟수(`missedCount`) ≥ 2이고 마지막 메시지 후 10분 이상 경과 시
- 캐릭터가 관계 레벨에 맞는 반응 생성 (AI 프롬프트에 지시)

## 5. 데이트 텐션 시스템

### 5.1 텐션 게이지

- **범위**: 0~100
- **표시**: 채팅방 헤더의 그라데이션 프로그레스 바 (핑크→로즈→핑크)
- **100 도달 시**: 하트 아이콘 펄스 애니메이션 + 데이트 버튼 표시

### 5.2 텐션 변화 요인

| 행동 | 텐션 변화 |
|------|-----------|
| 채팅 메시지 | AI 응답의 `\|\|TENSION:±n\|\|` |
| 선물 (치킨) | +10 |
| 선물 (명품 가방) | +30 |
| 피드 댓글 | +3 |
| 데이트 완료 | → 0으로 리셋 |
| 안읽음 누적 ≥ 3 | -5 |
| 읽씹 (1단계) | -3 |

### 5.3 텐션 변화 연출

**상승 시**: 핑크색 하트(Heart) 파티클이 아래에서 위로 떠오름

- 크기: 14~24px 랜덤
- 순차 딜레이: 0.15초 간격
- 애니메이션: `floatHeart` (1.2초, 위로 떠오르며 사라짐)

## 6. 선톡(트리거) 시스템

### 6.1 트리거 발동 조건

| 조건 | 설명 |
|------|------|
| 채팅방 미진입 | 현재 해당 캐릭터 채팅방을 보고 있지 않을 때 |
| 쿨다운 | 같은 캐릭터 연속 트리거 방지 (10초) |
| 안읽음 제한 | 미확인 메시지 10개 미만일 때만 발동 |
| 확률 판정 | 캐릭터별 `triggerChance` 확률 통과 |

### 6.2 트리거 발동 시점

1. **채팅방 나갈 때**: 나간 캐릭터를 제외한 다른 캐릭터들에게 확률 트리거
2. **5초 미입력**: 채팅방 내 5초간 입력 없으면 다른 캐릭터에게 확률 트리거
3. **스케줄 변경**: 60초 간격 체크, 활동 변경 감지 시 확률 트리거

### 6.3 메시지 선택 로직

놓침 횟수(`missedCount`)에 따른 3단계 메시지 풀:

| 단계 | 놓침 횟수 | 메시지 풀 |
|------|-----------|-----------|
| 첫 연락 | 0~1 | 스케줄 기반 메시지 (현재 활동 관련) |
| 리마인드 | 2~4 | 에스컬레이션 메시지 (짜증/걱정) |
| 화남 | 5+ | 에스컬레이션 메시지 (서운함/포기) |

놓침 ≥ 5일 경우 한 번에 1개만, 그 외 1~2개 메시지 전송.
복수 메시지는 800ms + 랜덤 0~400ms 간격으로 순차 전송.

### 6.4 토스트 알림

선톡 메시지 수신 시:
- 화면 상단 토스트 팝업 (캐릭터 아바타 + 메시지 미리보기)
- 최대 3개 스택 (오래된 것부터 제거)
- 3초 후 자동 소멸 (0.4초 페이드아웃 애니메이션)
- 클릭 시 해당 캐릭터 채팅방으로 이동
- 알림 사운드 재생

## 7. 스케줄 시스템

### 7.1 스케줄 구조

각 캐릭터는 24시간을 커버하는 6~7개 시간 슬롯을 보유.

```js
{
  start: Number,        // 시작 시간 (24시간제)
  end: Number,          // 종료 시간
  activity: String,     // 활동명
  messages: String[],   // 선톡 메시지 풀 (5~8개)
  feed: {               // 피드 포스트 데이터 (null이면 피드 없음)
    caption: String,
    chance: Number       // 발생 확률 (0.0~1.0)
  },
  statusCaptions: String[]  // 상태 피드 캡션 풀 (0~4개)
}
```

- 활동이 변경되면 `feed.chance` 확률로 피드 게시

## 8. 피드(인스타) 시스템

### 8.1 피드 구조

```js
{
  id: String,                // 고유 ID (예: "schedule-1234567890-ann")
  image: String,             // 포토카드 이미지
  caption: String,           // 게시글 캡션
  likes: Number,             // 좋아요 수 (랜덤 300~5500)
  timestamp: String,         // "방금"
  comments: Array,           // 다른 캐릭터 댓글 [{charId, text}]
  userComment: String?,      // 유저 댓글
  replies: Array?            // 유저 댓글에 대한 답글 [{charId, text}]
}
```

### 8.2 유저 댓글 상호작용

유저가 피드에 댓글을 달면:

1. **즉시 처리**
   - 유저 댓글 저장
   - 경험치 +10 (XP_REWARDS.feedLike)
   - 관계도 +8
   - 텐션 +3
   - 레벨업 체크

2. **피드 주인 캐릭터 답글**
   - LLM으로 유저 댓글에 대한 답글 생성
   - 피드에 답글로 추가

3. **다른 캐릭터 질투 반응** (각 캐릭터별 triggerChance 확률)
   - **댓글** (50% 확률): 관계 레벨 기반 질투/부러움 댓글
   - **DM** (60% 확률): 2~5초 딜레이 후 질투 DM 전송 + 토스트 알림

### 8.3 AI 댓글 생성

`generateFeedComment` 함수의 3가지 모드:

| 모드 | 조건 | 출력 |
|------|------|------|
| 일반 댓글 | 캡션만 제공 | 게시글에 대한 캐릭터 반응 |
| 유저 답글 | userComment 제공 | 유저 댓글에 대한 답글 |
| 질투 반응 | jealousyContext 제공 | 관계 레벨 기반 질투/부러움 (댓글 또는 DM) |

질투 레벨 (1~5): 무관심 → 약간 신경 쓰임 → 은근 질투 → 꽤 질투 → 매우 질투

### 8.4 피드 알림 시스템

새 피드 게시 시:
- `feedAlerts[charId]` 카운트 증가
- 채팅 홈 화면 스토리 원형에:
  - **그라데이션 테두리** (핑크→퍼플→핑크) 표시
  - **빨간 뱃지**에 새 게시글 수 표시
- 해당 캐릭터 피드 진입 시 알림 리셋

### 8.5 피드 정렬

- 알림이 있는 캐릭터 우선 (feedAlerts 카운트 내림차순)
- 동일 알림 수일 경우 최근 업데이트 순 (lastFeedAt 타임스탬프)

## 9. 선물 시스템

### 9.1 선물 종류

| 선물 | 시스템 메시지 | 텐션 | 관계도 | XP |
|------|--------------|------|--------|-----|
| 치킨 🍗 | "치킨 기프티콘을 보냈습니다." | +10 | +0 | +5 |
| 명품 가방 💎 | "명품 가방을 보냈습니다." | +30 | +0 | +5 |
| 아파트 🏢 | "강남 아파트를 선물했습니다!!!" | +0 | +20 | +5 |

### 9.2 선물 제한

## 14. UI/UX 연출

### 14.1 애니메이션 목록

| 이름 | 용도 | 동작 |
|------|------|------|
| `fadeIn` | 메시지/컴포넌트 진입 | 아래에서 위로 + 페이드인 (0.3초) |
| `slideUp` | 모달/패널 진입 | 아래에서 위로 슬라이드 (0.3초) |
| `slideDown` | 토스트 진입 | 위에서 아래로 슬라이드 (0.3초) |
| `toastOut` | 토스트 퇴장 | 위로 올라가며 페이드아웃 (0.4초) |
| `floatHeart` | 텐션 상승 | 하트가 위로 떠오르며 사라짐 (1.2초) |
| `heartBreak` | 텐션 감소 | 깨진 하트가 아래로 떨어짐 (1초) |

### 14.2 토스트 스택

- 최대 3개 동시 표시
- 아래 토스트일수록: `top` 오프셋 +8px, `scale` -3%, `opacity` -20%
- 퇴장 중인 토스트는 인라인 스타일 제거 (CSS 애니메이션과 충돌 방지)

### 14.3 채팅 홈 피드 알림

- 새 피드가 있는 캐릭터: 그라데이션 링 (핑크→퍼플→핑크)
- 빨간 뱃지에 새 게시글 수 표시
- 알림 없는 캐릭터: 회색 테두리

## 15. 화면 구성

### 15.1 API 키 입력 화면 (`ApiKeyScreen`)

- Gemini API 키 입력
- 입력 완료 시 채팅 목록으로 전환
- localStorage에 키 저장

### 15.2 채팅 홈 화면 (`ChatListScreen`)

- **상단**: "OshizTalk" 헤더 + 검색 아이콘
- **스토리 바**: 캐릭터 피드 원형 (클릭 → 피드 화면) + ME 프로필 버튼
- **채팅 목록**: 캐릭터별 아바타, 이름, 마지막 메시지, 시간, 안읽음 뱃지
  - 가장 최근 메시지 받은 캐릭터가 상단
  - 안읽음 메시지 있으면 초록 점 표시

### 15.3 채팅방 화면 (`ChatRoom`)

- **헤더**: 뒤로 버튼, 아바타, 이름, 관계 레벨 뱃지, 텐션 게이지
- **메시지 영역**: 스크롤 가능, 자동 스크롤
- **하트 파티클**: 텐션 변화 시 연출
- **데이트 버튼**: 텐션 100 도달 시 표시 (바운스 애니메이션)
- **입력 바**: 텍스트 입력 + 전송 버튼 + 선물 토글
- **선물 메뉴**: 치킨/명품가방/아파트 선택

### 15.4 피드 화면 (`FeedScreen`)

- **헤더**: 캐릭터 아바타, 이름, 뒤로 버튼
- **게시글**: 이미지, 캡션, 좋아요 수, 타임스탬프
- **댓글 영역**: 다른 캐릭터 댓글, 유저 댓글, 답글
- **댓글 입력**: 100자 제한, 좋아요 전 댓글 유도 넛지

### 15.5 프로필 화면 (`ProfileScreen`)

- **플레이어 카드**: 이름, 레벨 칭호, 설명, 좌우명
- **레벨 섹션**: XP 프로그레스 바, 현재/다음 XP
- **특성 태그**: 성격 태그 표시
- **관계 카드**: 각 캐릭터별 관계도/텐션 표시

### 15.6 상태 바 (`StatusBar`)

- iPhone 스타일 상태 바
- 실시간 시계 (1초 간격 업데이트)
- 시그널, WiFi, 배터리 아이콘
